────────────────────────────────────────────────────────────────────────────────
A. 周期广播报文
────────────────────────────────────────────────────────────────────────────────

  CAN ID : 0x3F1  (ECU → Bus)
  周期   : 1000 ms
  DLC    : 4 字节

  +--------+------+------------+---------------------------+
  | 字节   | 类型 | 信号       | 编码                      |
  +--------+------+------------+---------------------------+
  | [0]    | uint8| 总电压低字 | raw = V / 0.1             |
  | [1]    | uint8| 总电压高字 | 例: 48.0V → raw=480       |
  | [2]    | uint8| SOC 低字   | raw = SOC% / 0.1          |
  | [3]    | uint8| SOC 高字   | 例: 80.0% → raw=800       |
  +--------+------+------------+---------------------------+

────────────────────────────────────────────────────────────────────────────────
B. UDS 诊断服务 (请求 0x7E0/0x7DF → 响应 0x7E8)
────────────────────────────────────────────────────────────────────────────────

  协议格式: [PCI] [SID] [SubFunc/DID_H] [DID_L] [Data...]
  正响应  : [PCI] [SID+0x40] [Data...]
  负响应  : [PCI] [0x7F] [SID] [NRC]

  已实现 SID 列表:

  +------+-----------------------------+---------------------------------------+
  | SID  | 功能                        | 已支持内容                            |
  +------+-----------------------------+---------------------------------------+
  | 0x10 | 诊断会话控制                | sub=01 默认 / 02 编程 / 03 扩展      |
  | 0x11 | ECU 复位                    | sub=01 硬复位 / 03 软复位             |
  | 0x22 | 按 DID 读数据 (见下表)      | 12 个 DID                             |
  | 0x3E | Tester Present              | sub=00 需要响应 / 80 抑制响应         |
  +------+-----------------------------+---------------------------------------+

  已实现 DID 读取明细 (SID 0x22):

  请求: [0x22] [DID_H] [DID_L]
  响应: [0x62] [DID_H] [DID_L] [Data...]

  +--------+--------+-----------+------------------+---------------------------+
  | DID    | 信号   | 长度      | 类型 / 分辨率    | 说明                      |
  +--------+--------+-----------+------------------+---------------------------+
  | 0x2001 | 总电压 | 2B (BE)   | uint16 × 0.1V   | 例: raw=480 → 48.0V       |
  | 0x2002 | 电流   | 2B (BE)   | int16  × 0.1A   | 充电正 / 放电负           |
  | 0x2003 | SOC    | 1B        | uint8  × 0.5%   | 例: raw=160 → 80.0%       |
  | 0x2101 | 最高单体电压 | 2B (BE) | uint16 × 0.001V | 例: raw=4200 → 4.200V |
  | 0x2102 | 最低单体电压 | 2B (BE) | uint16 × 0.001V | 例: raw=3800 → 3.800V |
  | 0x2103 | 最高温度 | 1B      | uint8  offset+40 | raw=T+40, 例: 25°C→65   |
  | 0x2104 | 最低温度 | 1B      | uint8  offset+40 | 同上                     |
  | 0x3001 | BMS状态 | 1B       | 位域 (见下方)    | 系统模式+使能+均衡        |
  | 0x3002 | 充电状态 | 1B      | 0x00=禁用 0x01=充电中 |                     |
  | 0x3003 | 均衡状态 | 1B      | 0x00=未均衡 0x01=均衡中 |                   |
  | 0x3004 | 保护标志 | 2B (BE) | uint16 位掩码    | 见保护标志位表            |
  | 0x3005 | 故障码   | 2B (BE) | uint16 DTC 编码  | 见 DTC 映射表             |
  +--------+--------+-----------+------------------+---------------------------+

  DID 0x3001 BMS状态 位域定义:
  +--------+------+-------------------------------------------+
  | 位     | 信号 | 值                                        |
  +--------+------+-------------------------------------------+
  | [2:0]  | 系统模式 | 0=NULL 1=充电 2=放电 3=待机 4=睡眠  |
  | [3]    | 充电使能 | 1=允许充电                            |
  | [4]    | 放电使能 | 1=允许放电                            |
  | [5]    | 均衡中   | 1=正在均衡                            |
  | [7:6]  | 保留     |                                       |
  +--------+------+-------------------------------------------+

  DID 0x3004 保护标志位定义:
  +------+--------+------------------+
  | 位   | 缩写   | 含义             |
  +------+--------+------------------+
  | bit0 | OV     | 充电过压         |
  | bit1 | OCC    | 充电过流         |
  | bit2 | OTC    | 充电过温         |
  | bit3 | LTC    | 充电低温         |
  | bit4 | UV     | 放电欠压         |
  | bit5 | OCD    | 放电过流         |
  | bit6 | SCD    | 放电短路         |
  | bit7 | OTD    | 放电过温         |
  | bit8 | LTD    | 放电低温         |
  | 15~9 | 保留   |                  |
  +------+--------+------------------+

  DID 0x3005 DTC 故障码映射 (按优先级):
  +--------+------------------+------------------+
  | DTC    | 含义             | 触发标志位       |
  +--------+------------------+------------------+
  | 0x0203 | 放电短路         | SCD  (bit6)      |
  | 0x0202 | 放电过流         | OCD  (bit5)      |
  | 0x0201 | 充电过流         | OCC  (bit1)      |
  | 0x0101 | 充电过压         | OV   (bit0)      |
  | 0x0102 | 放电欠压         | UV   (bit4)      |
  | 0x0302 | 放电过温         | OTD  (bit7)      |
  | 0x0301 | 充电过温         | OTC  (bit2)      |
  | 0x0304 | 放电低温         | LTD  (bit8)      |
  | 0x0303 | 充电低温         | LTC  (bit3)      |
  | 0x0000 | 无故障           |                  |
  +--------+------------------+------------------+

================================================================================
一、项目通信分层
================================================================================

  ASW      : app.c / Can_Router.c / BMS 业务处理
  RTE      : Rte.c，周期调度 COM、DCM、CanTp
  SERVICES : Com.c 周期报文；Dcm.c UDS 服务；CanTp.c CAN-TP 传输层
  ECUAL    : CanIf.c，CAN 接口、收发缓存、邮箱
  MCAL     : CAN 驱动、中断、底层硬件发送/接收

  UDS 请求入口:
    CAN 驱动 -> CanIf -> Com_RxIndication -> Can_Router/Hook
             -> CanTp_RxIndication -> Dcm_ProcessRequest

  UDS 响应出口:
    Dcm_GetResponse -> CanTp_Transmit -> CanTp_GetTxFrame
                    -> CanIf_Transmit -> CAN 驱动发送


================================================================================
二、CAN 收发主流程
================================================================================

  接收 Rx:
    CAN RX 中断
      -> Can_MainFunction_Read()
      -> CanIf_RxIndication()
      -> CanIf 邮箱
      -> CanIf_MainFunction_Read()
      -> Com_RxIndication()
      -> 根据 CAN ID 分发到 UDS 或普通业务

  发送 Tx:
    周期报文或诊断响应
      -> Com / Dcm 生成数据
      -> CanIf_Transmit()
      -> Can_Write()
      -> CAN 总线

  当前周期报文:
    CAN ID : 0x3F1
    周期   : 1000 ms
    数据   : [Voltage_L, Voltage_H, SOC_L, SOC_H]

================================================================================
四、CAN-TP 单帧格式 ISO 15765-2
================================================================================

  本项目当前主要支持 Single Frame，适合 UDS payload 长度 1~7 字节。

  单帧格式:
    Byte0       : PCI = 0x0N，N 表示 UDS payload 长度
    Byte1~ByteN : UDS payload
    剩余字节    : 填充 0xAA

  示例 1: 诊断会话控制请求 10 01
    CAN ID : 0x7E0
    Data   : 02 10 01 AA AA AA AA AA
             |  |  |
             |  |  +-- SubFunction = 0x01 默认会话
             |  +----- SID = 0x10 Diagnostic Session Control
             +-------- PCI = 单帧，payload 长度 2

  示例 2: 正响应 50 01
    CAN ID : 0x7E8
    Data   : 02 50 01 AA AA AA AA AA

  多帧 FF/CF/FC:
    当前文档只列格式，不展开实现。
    当 UDS payload > 7 字节时，需要完整 CAN-TP 多帧收发状态机。

五、UDS 标准协议主要服务表 ISO 14229-1
================================================================================

  UDS 基本格式:
    请求   : [SID] [SubFunction 或 DID_H] [DID_L 或 Data...]
    正响应 : [SID + 0x40] [Data...]
    负响应 : [0x7F] [原 SID] [NRC]

  +------+-----------------------------+----------------------------+-------------------------+
  | SID  | 服务名                      | 请求数据                   | 正响应数据              |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x10 | Diagnostic Session Control  | 10 sub                     | 50 sub                  |
  |      | 诊断会话控制                | sub=01 默认会话            |                         |
  |      |                             | sub=02 编程会话            |                         |
  |      |                             | sub=03 扩展会话            |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x11 | ECU Reset                   | 11 resetType               | 51 resetType            |
  |      | ECU 复位                    | 01 硬复位                  |                         |
  |      |                             | 02 key off/on reset        |                         |
  |      |                             | 03 软复位                  |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x14 | Clear Diagnostic Info       | 14 group_H group_M group_L | 54                      |
  |      | 清除 DTC                    | 常用 FF FF FF 清全部       |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x19 | Read DTC Information        | 19 sub [mask/status...]    | 59 sub [DTC data...]    |
  |      | 读取 DTC                    | 常用 02 按状态掩码读 DTC   |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x22 | Read Data By Identifier     | 22 DID_H DID_L             | 62 DID_H DID_L data...  |
  |      | 按 DID 读数据               | 例如 22 F1 00              |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x27 | Security Access             | 27 sub [key...]            | 67 sub [seed...]        |
  |      | 安全访问                    | 奇数 sub 请求 seed         |                         |
  |      |                             | 偶数 sub 发送 key          |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x2E | Write Data By Identifier    | 2E DID_H DID_L data...     | 6E DID_H DID_L          |
  |      | 按 DID 写数据               | 写入指定 DID 数据          |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x31 | Routine Control             | 31 type RID_H RID_L ...    | 71 type RID_H RID_L ... |
  |      | 例程控制                    | type=01 start              |                         |
  |      |                             | type=02 stop               |                         |
  |      |                             | type=03 request result     |                         |
  +------+-----------------------------+----------------------------+-------------------------+
  | 0x3E | Tester Present              | 3E 00                      | 7E 00                   |
  |      | 测试仪保持在线              | 3E 80 表示抑制正响应       |                         |
  +------+-----------------------------+----------------------------+-------------------------+

  说明:
    sub/function byte 最高位 bit7 = suppressPosRspMsgIndicationBit。
    例如 10 83 表示进入扩展会话 0x03，但抑制正响应。

六、常用 NRC 否定响应码
================================================================================

  +------+------------------------------------+----------------------+
  | NRC  | 标准含义                           | 常见原因             |
  +------+------------------------------------+----------------------+
  | 0x10 | General Reject                     | 一般拒绝             |
  | 0x11 | Service Not Supported              | SID 不支持           |
  | 0x12 | SubFunction Not Supported          | 子功能不支持         |
  | 0x13 | Incorrect Message Length Or Format | 长度或格式错误       |
  | 0x22 | Conditions Not Correct             | 当前条件不满足       |
  | 0x24 | Request Sequence Error             | 请求顺序错误         |
  | 0x31 | Request Out Of Range               | DID/RID/参数越界     |
  | 0x33 | Security Access Denied             | 安全访问未解锁       |
  | 0x35 | Invalid Key                        | key 错误             |
  | 0x36 | Exceed Number Of Attempts          | 尝试次数超限         |
  | 0x37 | Required Time Delay Not Expired    | 等待时间未到         |
  +------+------------------------------------+----------------------+

七、UDS 调试示例
================================================================================

  1. 进入默认会话
     Tx: ID 0x7E0, Data 02 10 01 AA AA AA AA AA
     Rx: ID 0x7E8, Data 02 50 01 AA AA AA AA AA

  2. Tester Present 保持会话
     Tx: ID 0x7E0, Data 02 3E 00 AA AA AA AA AA
     Rx: ID 0x7E8, Data 02 7E 00 AA AA AA AA AA

  3. 读取 DID 0xF100
     Tx: ID 0x7E0, Data 03 22 F1 00 AA AA AA AA
     Rx: ID 0x7E8, Data NN 62 F1 00 ...

  4. 不支持的服务示例
     Tx: ID 0x7E0, Data 01 99 AA AA AA AA AA AA
     Rx: ID 0x7E8, Data 03 7F 99 11 AA AA AA AA

八、当前代码实现注意点
================================================================================

  1. CanTp 当前重点是单帧 SF，UDS payload 长度建议控制在 1~7 字节。
  2. Dcm 生成响应后，需要周期调用 Rte_MainFunction_Dcm() 才能真正发出响应。
  3. 非默认诊断会话需要 Tester Present 维持，否则 S3 超时后回默认会话。
  4. 功能寻址 0x7DF 通常不应回复某些否定响应，具体策略按项目需求处理。
  5. 若要支持长 DID 数据、DTC 列表、例程结果，需要补齐 CAN-TP 多帧。
